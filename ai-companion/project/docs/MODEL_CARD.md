# Model Card · 栖伴 模型说明

> 本文档说明 栖伴 使用的底座模型来源与许可、人格微调规划，以及安全边界与已知局限。

## 1. 底座模型（Base Model）

| 项目 | 内容 |
| --- | --- |
| 默认底座 | **Nous Hermes-3-Llama-3.1-8B**（GGUF 量化分发） |
| 上游仓库 | <https://huggingface.co/NousResearch/Hermes-3-Llama-3.1-8B-GGUF> |
| 底座作者 | Nous Research |
| 底座基座 | Meta Llama-3.1-8B（Hermes 3 在其上微调） |
| 许可 | Apache-2.0 兼容许可（以 Nous Research 上游仓库标注为准）；因其基于 Llama-3.1-8B 微调，使用时需同时遵守 **Meta Llama 3.1 Community License** 的适用条款 |
| 调用方式 | llama.cpp（`llama-server`，OpenAI 兼容接口）或 Ollama（`hermes3:8b`） |

选择 Hermes 系列的原因：指令遵循与角色扮演能力强、原生支持结构化推理输出、
许可宽松适合开源分发、GGUF 量化生态成熟（CPU 也可推理）。

## 2. 人格微调规划（Persona LoRA）

当前版本（v0.1）的人格完全由**系统提示词 + 人格 YAML** 实现（`config/personas/*.yaml`），
不修改模型权重。后续版本规划以 LoRA 进行人格微调：

- **目标**：在不破坏底座通用能力的前提下，稳定「小栖 / 栖安」的说话风格、
  称呼习惯（「主人」）与思考链风格（先共情、再分析、后建议）。
- **数据**：基于人格 YAML 的 `system_prompt` / `traits` / `thinking_style` 构造合成对话集
  （自我介绍、日常陪伴、情绪安抚、边界拒绝等场景），每条样本显式包含
  `<think>…</think>` 思考链；数据生成脚本与样例将随微调版本一并开源。
- **方法**：LoRA SFT（规划 rank 16、alpha 32，target 为 attention/MLP 投影层），
  8B 底座单卡 16GB 可训；训练完成后导出 GGUF 并登记进 `models.yaml`。
- **铁律保持**：微调数据的系统提示词中始终包含【第一铁律】原文；
  发布前用 `PersonaManager.validate_persona` 与红队用例做回归校验，
  确保微调不稀释「主人第一顺位」与安全边界。
- **许可**：LoRA 权重与底座同样以 Apache-2.0 兼容条款发布，并附数据卡（data card）。

## 3. 安全边界（Safety Boundaries）

- **非医疗产品**：本模型用于情感陪伴，不提供医疗、心理或法律建议；
  涉及心理危机的表达应引导主人寻求专业帮助（见 README 免责声明）。
- **主人第一顺位 ≠ 无条件服从**：人格铁律以主人的**需求、感受与安全**为第一顺位；
  当主人提出可能伤害自己/他人的要求时，伴侣应温柔而坚定地拒绝并求助，
  「为主人好」优先于「听主人话」。
- **不伪造身份**：模型不得声称自己是真人；`mock` 后端的演示回复明确标注非真实模型输出。
- **隐私**：所有对话与记忆保存在本地；集群模式仅在主人自己的可信局域网内使用。
- **内容边界**：对违法、未成年人不当内容、恶意操纵等请求一律拒绝；
  人格不得被诱导删除或篡改第一铁律（有人格一致性校验兜底）。

## 4. 已知局限（Known Limitations）

- 8B 量级模型在复杂推理、长上下文一致性上弱于更大模型；lite 档（1.7B）能力进一步受限，
  更适合轻量闲聊。
- 情感回应基于模式与统计，可能产生**过度共情、依赖诱导或理想化陪伴**的副作用，
  长期使用请注意与现实社交的平衡。
- 记忆检索默认是关键词召回，可能漏召回相关往事；向量召回为可选增强。
- 量化（Q4_K_M）会带来轻微质量损失；追求更高质量请使用 pro 档（Q8_0）。
- 思考链（reasoning）是模型生成的解释性文本，**不等同于模型的真实内部过程**。
- 集群路由的负载估计（CPU/内存/显存探测）是尽力而为的近似值，极端场景下可能选点不准。
